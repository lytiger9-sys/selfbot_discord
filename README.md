기능(function)폴더:
1. help폴더
    !help: 현재까지 있는 명령어 목록을 보여주는 파일이다. 임베드 형식(컴포넨트 v2)으로 현재까지의 명령어 내용을 알려주며, 명령어 카테고리는 파일명으로 한다. 즉, 라이센스를 가진 db에 등록된 유저가 !help를 하면 폴더명과 해당 폴더명의 각 기능을 목차 형식으로 보여준다. 설명이 두개 이상이더라도 그대로 명령어 개수만큼 상세설명을 한다.
        예시: 1. messages 명령어 목록
                1. !afk :상태를 지정하여 자리비움 상태에 언급시 정해진 메시지를 보내는 기능입니다.
                2. !파트너 메시지: ========
             2. info 명령어 목록
                1. !정보 유저id: 유저의 정보를 불러옵니다. 
                2. !--: =======
    이와 같은 형식으로 함. 각 파일에 명령어 이름과 설명을 적는 란을 만들어 놓을 건데, !help를 누르면 기존 파일 수정 없이도 계속 수정되도록.
2. info 폴더
    userinfo.js
    !정보 유저id: 유저의 해당 채널에 들어온 일자, 디스코드 가입 일자, ip등 얻을 수 잇는 모든 정보를 얻습니다.
3. auto-messages 폴더
    afk.js
    !afk: 메시지를 적을 수 있는 인풋란과 on/off토글을 만들어 현재 자리비움 상태와 자리비움시 보낼 메시지를 지정할 수 있게 한다.
    partner-messages.js
    !파트너 메시지 설정: 마지막으로 보낸 메시지를 파트너 메시지로 지정한다.
    !파트너 서버 설정: 파트너 서버의 홍보용 채널 id를 입력하면 해당 채널에 그 시간부로 하루에 한번씩 정해진 파트너 메시지를 보낸다.
    !파트너 서버 목록 보기: 지정된 채널의 서버 이름들을 나열하고, 서버 이름 옆에 삭제 버튼을 만들어 파트너 채널을 삭제한다. (질문은 두번 하기)
    !웹훅 (보내야할 메시지): 메시지 앞에 !웹훅을 입력하면 해당 메시지를 자신의 프로필을 한 웹훅으로 메시지를 보낸다.
    messagedelete.js
    !서버 메시지 삭제 (개수): 해당 서버에서 보낸 메시지를 최근순으로 정렬하여 개수만큼 삭제(서버가 아닌 dm이나 다른 공간이라면 작동 오류 메시지 출력 후 삭제.)
    !채널 메시지 삭제 (개수): 해당 채널에서 보낸 메시지를 최근순으로 정렬하여 개수만큼 삭제(dm삭제도 여기에 포함)
    !전체 메시지 삭제 (개수): 보낸 모든 메시지를 최근순으로 정렬하여 개수만큼 삭제


판매용(sales)폴더
1. masters.json
    봇 개발자에 대한 정보를 두는 곳.
    개발자의 서버 id, 토큰, 개발자 임명 시간, 이름을 저장하고, 관리자용 명령어를 실행할때 이 파일을 기준으로 함.
2. admin 풀더<-모든 명령어는 관리자만 실행할 수 있다, 라이센스는 MYSQL을 사용할 예정이다.
    deleteLicense.js
    !라이센스 삭제 (고유 id): 라이센스 목록에서 볼 수 있는 라이센스 고유 id를 입력하면 해당 라이센스는 삭제 처리하고, 삭제된 라이센스에 대한 정보를 해당 채널에 보내주기
    makeLicense.js
    !라이센스 생성 (기간)(개수): 해당 기간만큼 해당 개수의 라이센스 발급한다.
    showLicense.js
    !라이센스 현황: 현재까지의 모든 라이센스를 텍스트파일로 첨부한다.(너무 많아질것을 대비) 또한, 각 라이센스 생성 당시에 고유 id를 부여하여, 삭제를 용이하게 한다. 목록은 유저명, 라이센스내용, 만료기간(또는 미사용)
3. panels폴더<-모든 명령어는 관리자만 실행할 수 있다.
    buttons폴더(버튼클릭을 통해 유저가 사용할 수 있음, 해당 정보들은 유저가 다시 버튼을 눌러도 저장되도록)
        checkLicense.js: 라이센스가 유효한지 검증하는 파일.
        startlicense.js: 라이센스 입력 버튼을 만들고 이미 있던 미사용중인 발급된 라이센스와 비교하여 해당 유저에게 권한을 부여하는 파일. 만약 이미 라이센스가 있는 유저라면, 추가한 라이센스의 기간만큼 남은 일수를 추가한다.
        myprofile.js: 내정보 버튼 생성과 유저가 클릭시 자신의 라이센스 정보(만료일, 시작일 등)
        inputtoken.js: 자신의 토큰을 입력할 모달창을 만드는 파일. startlicense.js에서 라이센스 입력 시에 토큰을 입력할 모달창을 만드는 파일. 함께 db에 연결하여 유저id와 토큰, 라이센스정보(만료일 등)을 입력해주는 파일이다.
        status.js: 자신이 라이센스를 이미 입력한 상태라면, 시작일, 남은 만료일을 보여주는 내정보 버튼을 만드는 파일.
        streaming.js: 스트리밍 설정하기 버튼을 만든다. 00스트리밍 중에서 00을 입력할 모달창을 만들고, 해당 값을 받아 저장 후 해당 유저의 프로필에 00스트리밍 중으로 나오도록 해주는 파일.
        rpc.js: rpc 설정하기 버튼을 만든다. 00플레이 중에서 00을 입력할 모달창을 만들고 해당 값을 받아 저장 후 해당 유저의 프로필에 00플레이 중으로 나오도록 해주는 파일
        imageselecter.js: 위의 두 버튼을 실행할때 보일 사진(00플레이중, 00스트리밍중 시 왼쪽의 사진.)을 받는 사진 선택버튼을 만들고, 큰 사진과 작은 사진의 url을 입력할 수 있는 모달창을 만든다.
    embed폴더
        embedBuilders.js에서 위의 모든 버튼을 포함할 임베드를 만든다.
        !임베드게시: masters.json파일의 유저가 해당 명령어를 실행하면 지정해놓은 임베드를 게시한다.(컴포넨트 v2)
